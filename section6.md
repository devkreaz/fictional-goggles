# Розділ 6. Архітектура системи та залежності між підсистемами

Інформаційна система керування бібліотекою буде реалізована як багатошарова веб­додаткова архітектура, що містить наступні основні підсистеми:

1. **Інтерфейс користувача (UI)**
   - Веб-представлення для читачів, бібліотекарів та адміністраторів.
   - Розроблений на основі HTML/CSS,React.
   - Комунікація з серверною логікою через REST API.

2. **Серверна логіка (Back-end)**
   - Обробка запитів, реалізація бізнес-правил (реєстрація, видача, пошук тощо).
   - Реалізована на мові програмування (наприклад, Python/Java/Node.js).
   - Розділена на модулі: користувачі, каталог, позички, бронювання, звіти.

3. **База даних**
   - Реляційна база (PostgreSQL, MySQL) з таблицями для користувачів, книг, транзакцій тощо.
   - Забезпечує транзакційність та збереження історії.

4. **Сервіс сповіщень**
   - Модуль для відправки електронних листів або SMS.

5. **Адміністративний модуль**
   - Налаштування системних параметрів, резервні копії, моніторинг.
   - Доступ через окремий розділ UI та API з підвищеними правами.

6. **Сервіси інтеграції**
   - Підключення до зовнішніх каталогів, бібліографічних сервісів.
   - Імпорт/експорт даних у форматах MARC, CSV тощо.

### Залежності між підсистемами

- UI ↔ Back-end: усі дії користувачів ініціюють HTTP-запити; back-end повертає JSON.
- Back-end ↔ DBMS: back-end виконує SQL-запити для читання/запису.
- Back-end ↔ Сервіс сповіщень: асинхронні виклики (черги повідомлень).
- Адміністративний модуль використовує ті ж API, що і інші користувачі, з авторизацією.
- Сервіси інтеграції можуть працювати як частина back-end або як зовнішні скрипти, які взаємодіють з DBMS.
